## <a name="what-is-distributed-caching"></a><span data-ttu-id="ff7d3-112">O que é cache distribuído</span><span class="sxs-lookup"><span data-stu-id="ff7d3-112">What is distributed caching</span></span>

<span data-ttu-id="ff7d3-113">Um cache fornece acesso de alta taxa de transferência e baixa latência aos dados de aplicativos comumente acessados, armazenando os dados na memória.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-113">A cache provides high throughput, low-latency access to commonly accessed application data, by storing the data in memory.</span></span> <span data-ttu-id="ff7d3-114">Para um aplicativo em nuvem, o tipo mais útil de cache é o cache distribuído, o que significa que os dados não são armazenados na memória do servidor web individual, mas em outros recursos da nuvem, e os dados armazenados em cache são disponibilizados para todos os servidores web de um aplicativo (ou outros VMs em nuvem que são usados pelo aplicativo).</span><span class="sxs-lookup"><span data-stu-id="ff7d3-114">For a cloud app the most useful type of cache is distributed cache, which means the data is not stored on the individual web server's memory but on other cloud resources, and the cached data is made available to all of an application's web servers (or other cloud VMs that are used by the application).</span></span>

![diagrama mostrando vários servidores web acessando os mesmos servidores de cache](distributed-caching/_static/image1.png)

<span data-ttu-id="ff7d3-116">Quando o aplicativo é dimensionado adicionando ou removendo servidores, ou quando os servidores são substituídos devido a atualizações ou falhas, os dados armazenados em cache permanecem acessíveis a todos os servidores que executam o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-116">When the application scales by adding or removing servers, or when servers are replaced due to upgrades or faults, the cached data remains accessible to every server that runs the application.</span></span>

<span data-ttu-id="ff7d3-117">Ao evitar o acesso de dados de alta latência de um armazenamento de dados persistente, o cache pode melhorar drasticamente a capacidade de resposta do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-117">By avoiding the high latency data access of a persistent data store, caching can dramatically improve application responsiveness.</span></span> <span data-ttu-id="ff7d3-118">Por exemplo, recuperar dados do cache é muito mais rápido do que recuperá-los de um banco de dados relacional.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-118">For example, retrieving data from cache is much faster than retrieving it from a relational database.</span></span>

<span data-ttu-id="ff7d3-119">Um benefício lateral do cache é a redução do tráfego para o armazenamento de dados persistente, o que pode resultar em custos menores quando há taxas de saída de dados para o armazenamento de dados persistente.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-119">A side benefit of caching is reduced traffic to the persistent data store, which may result in lower costs when there are data egress charges for the persistent data store.</span></span>

## <a name="when-to-use-distributed-caching"></a><span data-ttu-id="ff7d3-120">Quando usar cache distribuído</span><span class="sxs-lookup"><span data-stu-id="ff7d3-120">When to use distributed caching</span></span>

<span data-ttu-id="ff7d3-121">O cache funciona melhor para cargas de trabalho de aplicativos que fazem mais leitura do que a escrita de dados, e quando o modelo de dados suporta a organização de chave/valor que você usa para armazenar e recuperar dados em cache.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-121">Caching works best for application workloads that do more reading than writing of data, and when the data model supports the key/value organization that you use to store and retrieve data in cache.</span></span> <span data-ttu-id="ff7d3-122">Também é mais útil quando os usuários de aplicativos compartilham muitos dados comuns; por exemplo, o cache não forneceria tantos benefícios se cada usuário normalmente recuperasse dados exclusivos desse usuário.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-122">It's also more useful when application users share a lot of common data; for example, cache would not provide as many benefits if each user typically retrieves data unique to that user.</span></span> <span data-ttu-id="ff7d3-123">Um exemplo em que o cache pode ser muito benéfico é um catálogo de produtos, porque os dados não mudam com freqüência, e todos os clientes estão olhando para os mesmos dados.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-123">An example where caching could be very beneficial is a product catalog, because the data does not change frequently, and all customers are looking at the same data.</span></span>

<span data-ttu-id="ff7d3-124">O benefício do cache torna-se cada vez mais mensurável quanto mais uma escala de aplicativo, à medida que os limites de rendimento e atrasos de latência do armazenamento de dados persistente se tornam mais um limite para o desempenho geral do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-124">The benefit of caching becomes increasingly measurable the more an application scales, as the throughput limits and latency delays of the persistent data store become more of a limit on overall application performance.</span></span> <span data-ttu-id="ff7d3-125">No entanto, você pode implementar cache por outras razões além do desempenho também.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-125">However, you might implement caching for other reasons than performance as well.</span></span> <span data-ttu-id="ff7d3-126">Para dados que não precisem estar perfeitamente atualizados quando mostrados ao usuário, o acesso ao cache pode servir como um disjuntor para quando o armazenamento de dados persistente estiver sem resposta ou indisponível.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-126">For data that doesn't have to be perfectly up-to-date when shown to the user, cache access can serve as a circuit breaker for when the persistent data store is unresponsive or unavailable.</span></span>

## <a name="popular-cache-population-strategies"></a><span data-ttu-id="ff7d3-127">Estratégias populares de população de cache</span><span class="sxs-lookup"><span data-stu-id="ff7d3-127">Popular cache population strategies</span></span>

<span data-ttu-id="ff7d3-128">Para poder recuperar dados do cache, você tem que armazená-los lá primeiro.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-128">In order to be able to retrieve data from cache, you have to store it there first.</span></span> <span data-ttu-id="ff7d3-129">Existem várias estratégias para obter dados que você precisa em um cache:</span><span class="sxs-lookup"><span data-stu-id="ff7d3-129">There are several strategies for getting data that you need into a cache:</span></span>

- <span data-ttu-id="ff7d3-130">À vista sobre demanda / cache</span><span class="sxs-lookup"><span data-stu-id="ff7d3-130">On Demand / Cache Aside</span></span>

    <span data-ttu-id="ff7d3-131">O aplicativo tenta recuperar dados do cache, e quando o cache não tem os dados (uma "falha"), o aplicativo armazena os dados no cache para que eles estejam disponíveis na próxima vez.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-131">The application tries to retrieve data from cache, and when the cache doesn't have the data (a "miss"), the application stores the data in the cache so that it will be available the next time.</span></span> <span data-ttu-id="ff7d3-132">Da próxima vez que o aplicativo tentar obter os mesmos dados, ele encontra o que está procurando no cache (um "hit").</span><span class="sxs-lookup"><span data-stu-id="ff7d3-132">The next time the application tries to get the same data, it finds what it's looking for in the cache (a "hit").</span></span> <span data-ttu-id="ff7d3-133">Para evitar a busca de dados armazenados em cache que foram alterados no banco de dados, você invalida o cache ao fazer alterações no armazenamento de dados.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-133">To prevent fetching cached data that has changed on the database, you invalidate the cache when making changes to the data store.</span></span>
- <span data-ttu-id="ff7d3-134">Push de dados de fundo</span><span class="sxs-lookup"><span data-stu-id="ff7d3-134">Background Data Push</span></span>

    <span data-ttu-id="ff7d3-135">Serviços de fundo empurram dados para o cache em um horário regular, e o aplicativo sempre puxa do cache.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-135">Background services push data into the cache on a regular schedule, and the app always pulls from the cache.</span></span> <span data-ttu-id="ff7d3-136">Essa abordagem funciona muito bem com fontes de dados de alta latência que não exigem que você sempre retorne os dados mais recentes.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-136">This approach works great with high latency data sources that don't require you always return the latest data.</span></span>
- <span data-ttu-id="ff7d3-137">Disjuntor</span><span class="sxs-lookup"><span data-stu-id="ff7d3-137">Circuit Breaker</span></span>

    <span data-ttu-id="ff7d3-138">O aplicativo normalmente se comunica diretamente com o armazenamento de dados persistente, mas quando o armazenamento de dados persistente tem problemas de disponibilidade, o aplicativo recupera dados do cache.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-138">The application normally communicates directly with the persistent data store, but when the persistent data store has availability problems, the application retrieves data from cache.</span></span> <span data-ttu-id="ff7d3-139">Os dados podem ter sido colocados em cache usando o cache de lado ou a estratégia de pressão de dados em segundo plano.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-139">Data may have been put in cache using either the cache aside or background data push strategy.</span></span> <span data-ttu-id="ff7d3-140">Trata-se de uma estratégia de manipulação de falhas em vez de uma estratégia de melhoria de desempenho.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-140">This is a fault handling strategy rather than a performance enhancing strategy.</span></span>

<span data-ttu-id="ff7d3-141">Para manter os dados na corrente de cache, você pode excluir entradas de cache relacionadas quando seu aplicativo cria, atualiza ou exclui dados.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-141">In order to keep data in the cache current, you can delete related cache entries when your application creates, updates, or deletes data.</span></span> <span data-ttu-id="ff7d3-142">Se estiver tudo bem para o seu aplicativo às vezes obter dados ligeiramente desatualizados, você pode contar com um tempo de expiração configurável para definir um limite de quão antigos os dados de cache podem ser.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-142">If it's alright for your application to sometimes get data that is slightly out-of-date, you can rely on a configurable expiration time to set a limit on how old cache data can be.</span></span>

<span data-ttu-id="ff7d3-143">Você pode configurar expiração absoluta (quantidade de tempo desde que o item de cache foi criado) ou expiração deslizante (quantidade de tempo desde a última vez que um item de cache foi acessado).</span><span class="sxs-lookup"><span data-stu-id="ff7d3-143">You can configure absolute expiration (amount of time since the cache item was created) or sliding expiration (amount of time since the last time a cache item was accessed).</span></span> <span data-ttu-id="ff7d3-144">A expiração absoluta é usada quando você está dependendo do mecanismo de expiração do cache para evitar que os dados fiquem muito obsoletos.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-144">Absolute expiration is used when you are depending on the cache expiration mechanism to prevent the data from becoming too stale.</span></span> <span data-ttu-id="ff7d3-145">No aplicativo Fix It, expulsaremos manualmente itens de cache velhos e usaremos o vencimento deslizante para manter os dados mais atuais em cache.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-145">In the Fix It app, we'll manually evict stale cache items and we'll use sliding expiration to keep the most current data in cache.</span></span> <span data-ttu-id="ff7d3-146">Independentemente da política de expiração escolhida, o cache despejará automaticamente os itens mais antigos (Menos Recentemente Usados ou LRU) quando o limite de memória do cache for atingido.</span><span class="sxs-lookup"><span data-stu-id="ff7d3-146">Regardless of the expiration policy you choose, the cache will automatically evict the oldest (Least Recently Used or LRU) items when the cache's memory limit is reached.</span></span>
