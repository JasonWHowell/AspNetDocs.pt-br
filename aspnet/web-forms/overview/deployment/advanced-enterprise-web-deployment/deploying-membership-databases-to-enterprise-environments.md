## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="bb391-111">Quais são os problemas quando você implanta um banco de dados de associação?</span><span class="sxs-lookup"><span data-stu-id="bb391-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="bb391-112">Na maioria dos casos, quando você planeja uma estratégia de implantação para um banco de dados, a primeira coisa que você precisa considerar é qual dado você deseja implantar.</span><span class="sxs-lookup"><span data-stu-id="bb391-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="bb391-113">Em um ambiente de desenvolvimento ou teste, talvez você queira implantar dados de conta de usuário para facilitar testes rápidos e fáceis.</span><span class="sxs-lookup"><span data-stu-id="bb391-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="bb391-114">Em um ambiente de preparo ou de produção, é muito improvável que você queira implantar dados de conta de usuário.</span><span class="sxs-lookup"><span data-stu-id="bb391-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="bb391-115">Infelizmente, os bancos de dados de associação do ASP.NET introduzem alguns desafios específicos que tornam essa decisão muito mais complexa:</span><span class="sxs-lookup"><span data-stu-id="bb391-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="bb391-116">Uma implantação somente de esquema deixará o banco de dados de associação em um estado não operacional.</span><span class="sxs-lookup"><span data-stu-id="bb391-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="bb391-117">Isso se deve ao fato de o banco de dados de associação incluir algum dado de configuração (na tabela **aspnet\_SchemaVersions** ) que o Database requer para funcionar.</span><span class="sxs-lookup"><span data-stu-id="bb391-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="bb391-118">Dessa forma, se você executar uma implantação somente de esquema de seu banco de dados de associação para excluir a conta de usuário, precisará executar um script pós-implantação para adicionar os dados de configuração essenciais.</span><span class="sxs-lookup"><span data-stu-id="bb391-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="bb391-119">Dependendo de como o banco de dados de associação é configurado, o provedor de associação pode usar a chave do computador para criptografar senhas e armazená-las no banco de dados.</span><span class="sxs-lookup"><span data-stu-id="bb391-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="bb391-120">Nesse caso, qualquer dado de conta de usuário que você implantar com o banco de dados ficará inutilizável no servidor de destino.</span><span class="sxs-lookup"><span data-stu-id="bb391-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="bb391-121">Por esse motivo, a implantação de dados de conta de usuário não é um cenário com suporte.</span><span class="sxs-lookup"><span data-stu-id="bb391-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="bb391-122">Escolhendo uma estratégia de banco de dados de associação</span><span class="sxs-lookup"><span data-stu-id="bb391-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="bb391-123">Use estas diretrizes quando escolher como provisionar um banco de dados de associação em um ambiente de servidor corporativo:</span><span class="sxs-lookup"><span data-stu-id="bb391-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="bb391-124">Sempre que possível, não implante bancos de dados de associação.</span><span class="sxs-lookup"><span data-stu-id="bb391-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="bb391-125">Em vez disso, crie o banco de dados de associação manualmente no servidor de banco de dados de destino.</span><span class="sxs-lookup"><span data-stu-id="bb391-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="bb391-126">Se você não tiver personalizado seu esquema de banco de dados de associação, poderá simplesmente criar um novo no in-situ no destino usando a [ferramenta de registro de SQL Server do ASP.net (aspnet\_RegSql. exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="bb391-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="bb391-127">Se você não tiver nenhuma opção, mas implantar um banco&#x2014;de dados de associação, por exemplo, se tiver feito modificações extensivas no esquema&#x2014;de banco de dados, você deverá executar uma implantação somente de esquema do banco de dados de associação, excluir e executar um script de pós-implantação para adicionar quaisquer dados de configuração necessários.</span><span class="sxs-lookup"><span data-stu-id="bb391-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="bb391-128">Você pode encontrar diretrizes amplas sobre essas abordagens em [como implantar o banco de dados de associação do ASP.NET sem incluir contas de usuário](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="bb391-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="bb391-129">É importante lembrar que *o esquema do seu banco de dados de associação provavelmente será razoavelmente estático*.</span><span class="sxs-lookup"><span data-stu-id="bb391-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="bb391-130">Mesmo que você tenha personalizado o banco de dados de associação, é improvável que você precise atualizar o esquema regularmente,&#x2014;ele não será alterado com a mesma frequência que o código em um aplicativo Web ou um projeto de banco de dados.</span><span class="sxs-lookup"><span data-stu-id="bb391-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="bb391-131">Dessa forma, você não precisará incluir o banco de dados de associação em nenhum processo de implantação automatizado ou de etapa única.</span><span class="sxs-lookup"><span data-stu-id="bb391-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="bb391-132">Usando VSDBCMD para atualizar um esquema de banco de dados de associação</span><span class="sxs-lookup"><span data-stu-id="bb391-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="bb391-133">Se você modificar a estrutura do banco de dados de associação após sua primeira implantação, talvez não queira usar a Implantação da Web ferramenta de implantação da Web do Serviços de Informações da Internet (IIS) para reimplantar o banco de dados.</span><span class="sxs-lookup"><span data-stu-id="bb391-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="bb391-134">A funcionalidade de implantação de banco de dados no Implantação da Web não inclui a capacidade de fazer atualizações diferenciais em um banco de dados&#x2014;de destino, implantação da Web deve descartar e recriar o banco de dados.</span><span class="sxs-lookup"><span data-stu-id="bb391-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="bb391-135">Isso significa que você perde qualquer dado de conta de usuário existente, que normalmente é indesejável em ambientes de preparo ou de produção.</span><span class="sxs-lookup"><span data-stu-id="bb391-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="bb391-136">A alternativa é usar o utilitário VSDBCMD para atualizar o esquema do banco de dados de destino.</span><span class="sxs-lookup"><span data-stu-id="bb391-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="bb391-137">O VSDBCMD inclui dois recursos importantes.</span><span class="sxs-lookup"><span data-stu-id="bb391-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="bb391-138">Primeiro, ele pode importar o esquema de um banco de dados existente para um arquivo. dbschema.</span><span class="sxs-lookup"><span data-stu-id="bb391-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="bb391-139">Em segundo lugar, ele pode implantar um arquivo. dbschema em um banco de dados existente como uma atualização diferencial, o que significa que ele apenas faz as alterações necessárias para atualizar o banco de dados de destino e você não perde nenhum dado.</span><span class="sxs-lookup"><span data-stu-id="bb391-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

1. <span data-ttu-id="bb391-141">Use a ação de **importação** VSDBCMD para gerar um arquivo. dbschema para seu banco de dados de associação de origem.</span><span class="sxs-lookup"><span data-stu-id="bb391-141">Use the VSDBCMD **Import** action to generate a .dbschema file for your source membership database.</span></span> <span data-ttu-id="bb391-142">Esse procedimento é descrito em [como importar um esquema de um prompt de comando](https://msdn.microsoft.com/library/dd172135.aspx).</span><span class="sxs-lookup"><span data-stu-id="bb391-142">This procedure is described in [How to: Import a Schema from a Command Prompt](https://msdn.microsoft.com/library/dd172135.aspx).</span></span>
2. <span data-ttu-id="bb391-143">Use a ação de **implantação** VSDBCMD para implantar o arquivo. dbschema no banco de dados de associação de destino.</span><span class="sxs-lookup"><span data-stu-id="bb391-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="bb391-144">Esse procedimento é descrito em [referência de linha de comando para VSDBCMD. EXE (implantação e importação de esquema)](https://msdn.microsoft.com/library/dd193283.aspx).</span><span class="sxs-lookup"><span data-stu-id="bb391-144">This procedure is described in [Command-Line Reference for VSDBCMD.EXE (Deployment and Schema Import)](https://msdn.microsoft.com/library/dd193283.aspx).</span></span>
